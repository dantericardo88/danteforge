# Support

DanteForge maintainers triage new issues within 3 business days when reports include enough context to reproduce or route the work.

## Where To Ask

- Use bug reports for reproducible command failures, crashes, incorrect output, or broken generated artifacts.
- Use feature requests for workflow, command, assistant, or integration proposals.
- Use GitHub discussions for setup questions, workflow comparisons, and design tradeoffs before filing an implementation issue.
- Use `SECURITY.md` for private vulnerability reports.

## Useful Report Details

Include the exact command, expected behavior, actual behavior, reproduction steps, Node version, operating system, and logs with secrets removed.

For local setup failures, run:

```bash
danteforge doctor
danteforge --help
```

For verification failures, include the narrow command that failed and the first actionable error line.
