## 1. Structured Audit Logging (SIEM Integration)
- Add src/core/structured-audit.ts (461 lines)
- JSONL format: .danteforge/audit/detailed.jsonl
- Correlation IDs for distributed tracing
- Session IDs for CLI invocation grouping
- Event types: command_start, llm_call, file_write, git_operation, mcp_call, gate_check, error
- Fluent API: auditEvent('llm_call').provider('ollama').tokens(1234).log()
- Helper functions: logCommandStart, logLLMCall, logFileWrite, logGitOperation, logMCPCall, logGateCheck, logError
- Best-effort logging (never crashes operations)
- Ready for Splunk, ELK, Datadog integration
- Tests: tests/structured-audit.test.ts (57 tests, 100% coverage)

## 2. Universal Circuit Breaker (All I/O Operations)
- Add src/core/resilience.ts (352 lines)
- Circuit breaker for file I/O, Git, MCP, network (not just LLM)
- Timeout handling (default 5min, env: DANTEFORGE_OPERATION_TIMEOUT_MS)
- Retry logic with exponential backoff
- Concurrency limiting per operation type
- State machine: CLOSED → OPEN → HALF_OPEN
- Helper functions: readFileResilient, writeFileResilient, gitOperationResilient, mcpCallResilient, networkRequestResilient
- Error types: CircuitOpenError, OperationTimeoutError, ConcurrencyLimitError
- Prevents cascading failures and hung operations
- Tests: tests/resilience.test.ts (54 tests, 100% coverage)

## 3. SBOM Generation & Validation (Supply Chain Security)
- Add scripts/generate-sbom.mjs (183 lines)
- Add scripts/validate-sbom.mjs (221 lines)

- Human-readable summary generation
- Ready for Dependency-Track, Grype, Trivy scanning
