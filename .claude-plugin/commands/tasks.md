# /danteforge-tasks — Task Management

When the user invokes `/danteforge-tasks`, display the current task list with statuses and suggest the next action.

## Execution

```
danteforge tasks             # show current phase tasks
danteforge tasks --all       # show all phases
danteforge tasks --next      # show only the next task to work on
```

## What It Shows

```
Phase 2 of 3 — Core Implementation
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
✓  1. Add input validation module          [done]
▶  2. Implement rate limiter               [in-progress]
○  3. Wire rate limiter into MCP server   [pending]
○  4. Write security tests                [pending]

Next: task 2 is in progress — run /danteforge-forge to continue
```

## Task States

- `done` — completed and verified
- `in-progress` — currently being worked on
- `pending` — not started yet
- `blocked` — waiting on a dependency

## Workflow Context

Tasks are generated by `/danteforge-plan` and tracked in `.danteforge/TASKS.md` + `STATE.yaml`. Each forge run advances the task cursor.

## Advancing Tasks

To work on the next task:
```
danteforge forge   # picks up where tasks left off
```

CLI parity: `danteforge tasks [--all] [--next]`
